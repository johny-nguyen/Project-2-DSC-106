Proposition: Racial Discrimination is Rampant; Clearly underlined by NYPD complaints

FOR the Proposition

Design Decisions and Rationale:

  • Transforming the data to show complaints per 1000 residents

    • Score: 1.5

    Rationale: Instead of showing raw numbers, this transformation allows for a more reliable comparison between ethnic groups. We normalize against each group's population size by accounting for the fact that each ethnic group differ in size. This makes the visualization more persuasive by plotting the data along the same scale, allowing people to observe the differences especially between Black/Hispanic people versus Asian/White people. One possible limitation is that the transformation can also be misleading if the reader is not familiar with the scale. For instance, they might interpret a data point as 1 complaint rather than 1 complaints per 1000 residents.

  • Sequential Color Scheme

    • Score: 1

    Rationale: Using a sequential color scheme helps to visually represent the data in a logical order, making it easier for viewers to distinguish which ethnic group is the most affected. This makes the visualization more persuasive because the bolder colors draw more attention toward Black and Hispanic groups. Additionally, the red colors are used to highlight and make the line seem more concerning to viewers. One possible limitation is those with colorblindness might be prone to misinterpreting the colors. We also considered to use a different color scheme based on ethnicity, but it would be reinforcing dangerous stereotypes, so we opted to use darker colors and have it become lighter.

  • Line Plot on Four Ethnic Groups

    • Score: 2

    Rationale: By plotting using a line plot, we are able to effectively persuade viewers by showing the increase of complaints between ethnicities over time. We chose to only plot based on four of the Ethnic Groups because the other groups didn't particularly possess any meaningful information that would persuade users in a certain way. The other groups included "Unknown", "Other Race", "Refused", and "American Indian". For the first three groups, we can't really draw any conclusions about them since we don't know their ethnic group is and it would take away from our original proposition. For the "American Indian" group, the amount of complaints that were submitted were neglible as they had a very low number of complaints which may be seen as selective or "cherry picking". These changes make our visualization more persuasive since we can keep the focus on the more meaningful data points. As for alternative marks, we considered using a bar chart, but this would not be as effective since we were working with complaints reported over time.

  • Used data on all complaint types, rather than just certain types

    • Score: 1

    Rationale: We chose to use data on all complaint types. This makes our visualization more persuasive because we can see the full scope of complaints over time making the trend more noticeable. One limitation might be that the visualization can be seen as broad making the viewer unable to make a reasonable conclusion about the rise in complaints. One alternative we considered was filtering the data by the complaint type "Abuse of Authority" since it helps the viewer make the connection that maybe the complaints might be a result of a race profiling problem. Additionally, it allows us to potentially narrow down the problem to a specific type of complaint which makes it more insightful to the viewer. However, We chose not to filter by this complaint type because only specificing one type of complaint disregards the other complaint types that might also play a role in the grand scheme of things.

AGAINST the Proposition

Design Decisions and Rationale:

  • Normalized each row to 100%.

    Score: -2

    Rationale: By normalizing each row to 100 That erases the fact that Black New Yorkers file roughly 54% of complaints in absolute terms. Four near-identical bars let the reader conclude that everyone gets treated the same. This is deceptive because it hides the fact that Black New Yorkers are disproportionately affected by police misconduct, and it is not persuasive because it fails to highlight the disparities in complaint outcomes.

  • Three shades of the same blue.

    Score: -1

    Rationale: "Substantiated" and "Unsubstantiated" end up looking visually similar, which softens the contrast between them and makes it harder for viewers to quickly grasp the differences in complaint outcomes across racial groups. This is deceptive because it downplays the severity of the issue and is not persuasive because it fails to draw attention to the disparities in complaint outcomes across racial groups.

  • Comparison with a 100% Stacked Bar Chart

    Score: 1

    Rationale: A 100% stacked bar chart is reasonable since the goal is to compare outcome composition across groups. This is not deceptive because it accurately represents the proportion of each outcome category within each racial group, and it is somewhat persuasive because it allows viewers to see the minimal differences in complaint outcomes across racial groups.

Final Reflection

Working on this project with the NYPD civilian complaints dataset made it clear that data visualization is far from neutral. Using the same dataset, we were able to construct two convincing but opposing narratives about racial discrimination purely through design choices. While identifying trends across ethnic groups was relatively straightforward, the more difficult task was intentionally designing visualizations that both emphasized and downplayed those disparities. This required carefully applying transformations such as normalization, filtering, grouping, and visual emphasis to shift interpretation without altering the underlying data.

One of the most surprising aspects of this process was how impactful relatively simple techniques were. In the "FOR" visualization, converting raw counts into per-capita values and emphasizing certain groups through color and line thickness made disparities appear more pronounced and persistent. In contrast, in the "AGAINST" visualization, normalizing each group to 100%, grouping multiple complaint outcomes into broader categories, and using a uniform color palette made distributions appear nearly identical across ethnicities. Although both visualizations relied on the same dataset, these structural choices significantly changed which patterns were visible and how viewers interpreted them.

This project reshaped our understanding of ethical visualization by highlighting how easily persuasion can blur into deception. Ethical analysis is not only about avoiding incorrect data, but also about being transparent in how design choices influence perception. Persuasive techniques such as normalization and aggregation can be appropriate when they clarify comparisons, but they become misleading when they obscure important context, hide differences in scale, or remove meaningful detail. In our case, the same methods that clarified disparities in one visualization helped conceal them in another. This suggests that the boundary between acceptable persuasion and misleading design is not defined by the technique itself, but by whether it preserves or distorts the viewer's ability to understand the full context of the data.
